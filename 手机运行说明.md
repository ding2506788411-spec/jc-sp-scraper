# 手机端获取竞彩足球SP数据：云端运行方案

你没有电脑时，不建议在手机本机硬跑 Playwright/Chromium。正确做法是：手机只负责点按钮，真正的浏览器抓取在云端运行。

## 方案A：Google Colab，最适合马上测试

1. 手机浏览器打开 Colab：colab.research.google.com
2. 新建 Notebook，或者上传本包里的 `mobile_colab_jc_sp_runner.ipynb`
3. 把本包里的 `jc_sp_agent_browser_scraper.py` 上传到 Colab 左侧文件区
4. 依次运行 Notebook 单元格
5. 修改日期后运行抓取
6. 下载生成的 CSV

适合：临时抓 3 天、30 天、90 天。
缺点：需要手动点运行，不能保证后台一直运行。

## 方案B：GitHub Actions，适合每天自动跑

1. 手机打开 GitHub，新建一个私有仓库
2. 把本包全部上传到仓库根目录，注意 `.github/workflows/jc_sp_daily.yml` 也要上传
3. 进入仓库的 Actions 页面
4. 选择 `JC SP Daily Scraper`
5. 点 `Run workflow`
6. 输入 start_date / end_date，或者不填自动抓最近日期
7. 跑完后在 Artifacts 下载 CSV

适合：不用天天抱电脑，手机点一次，之后可以定时跑。

## 重要说明

- 不要高速抓取。建议每页间隔 2-7 秒。
- 如果抓不到，查看 `debug_pages` 里的 HTML/PNG。
- 本脚本只抓公开网页可见数据，不处理登录、验证码、破解接口、绕过权限。
